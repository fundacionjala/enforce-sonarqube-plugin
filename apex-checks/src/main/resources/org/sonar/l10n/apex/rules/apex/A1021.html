The "@Test" annotation is used to indicate when a class represents a test class, so it should be used only for classes (not enums or interfaces) and said class should be either public or private. Also, if a class is named with the word "test" it suggests this is a test class, so it should either have the @isTest annotation or be renamed.

Noncompliant Code Example

private class MyTest { _// missing annotation @isTest._ } @isTest class MyTest { _// missing modifier access._ } @isTest public enum MyTest { _//@isTest annotation should not be used for enums_ } @isTest public interface MyTest { _//@isTest annotation should not be used for interfaces_ }

Compliant Solution

@isTest private class MyTest { ... } @isTest public class MyTest {     public void testMyTest() {       // some assertion.     } }
