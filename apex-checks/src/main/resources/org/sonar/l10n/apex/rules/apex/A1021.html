The "@Test" annotation is used to indicate when a class represents a test class, so it should be used only for classes (not enums or interfaces) and test classes should be either public or private. Classes named with the word "test" should have the @isTest annotation or be renamed to exclude the word "test".

Noncompliant Code Example

private class MyTest {     // missing annotation @isTest.
}
@isTest
class MyTest {    // missing modifier access.
}

@isTest
public enum MyTest { //@isTest annotation should not be used for enums.
}

@isTest
public interface MyTest { //@isTest annotation should not be used for interfaces.
}

Compliant Solution

@isTest
private class MyTest {
...
}

@isTest
public class MyTest  {
    public void testMyTest() {
      ...
    }
}
